Telar Compositor — Terms of Service and Privacy Policy

Last updated: 15 March 2026

The Telar Compositor is a free, open-source tool developed by the UCSB Archives, Memory, and Preservation Lab and Neogranadina. It is provided as-is, for academic and scholarly use.

What the Compositor does

The Compositor is a visual editor for building Telar stories. It connects to your GitHub repository via a GitHub App, lets you edit content in a browser, and publishes changes back to your repo as commits attributed to your GitHub account.

Your content is yours

Everything you create in the Compositor — stories, object metadata, glossary entries — belongs to you. Published content lives in your own GitHub repository. The Compositor is a tool for editing and publishing, not a hosting platform.

What we store

  • GitHub tokens: When you sign in, we store an encrypted access token and refresh token in a Cloudflare D1 database. These are used solely to read and write to repositories you have authorized. Tokens are encrypted at rest using AES-256-GCM.
  • Draft content: While you are editing, your work-in-progress is stored in D1 as a working copy. This is a temporary workspace — your authoritative content lives in your GitHub repository.
  • Locale preference: A cookie stores your language preference (English or Spanish). It contains no personal information.

What we do not store or collect

  • We do not collect analytics or usage data.
  • We do not use tracking cookies or third-party trackers.
  • We do not store your GitHub password. Authentication is handled entirely by GitHub's OAuth flow.
  • We do not share your data with third parties.
  • We do not access repositories you have not explicitly authorized.

Infrastructure

The Compositor runs on Cloudflare Workers. Data is stored in Cloudflare D1 (SQLite). All connections use HTTPS. The source code is available at github.com/UCSB-AMPLab/telar-compositor.

Account and data deletion

You can revoke the Compositor's access to your GitHub repositories at any time via your GitHub settings. To request deletion of your stored data (tokens and draft content), contact contacto@neogranadina.org. Your published content in your GitHub repository is unaffected.
